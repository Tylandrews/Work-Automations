# Reporting (Weekly + Monthly)

This app can generate **weekly (Fridays)** and **monthly (end-of-month)** reporting snapshots in Supabase.

Snapshots are stored in `public.call_reports` and generated by the Supabase Edge Function `generate-reports`.

## What gets stored

Each row represents one reporting period for either:
- **team**: one row for the whole team
- **user**: one row per `user_id` that had calls in the period

The `metrics` JSON includes:
- `calls_total`
- `orgs_unique`
- `per_user_counts` (team rows only)
- `per_org_counts`
- `daily_timeseries` (array of `{ day, calls }`)

## Scheduling (Supabase Cron)

Create a scheduled invocation for the Edge Function in Supabase so it runs even if nobody opens the app.

Recommended schedule:
- **Daily**, late in the day in your preferred timezone (so month-end runs occur on the actual last calendar day).
- Example: `55 23 * * *` (23:55 each day)

In Supabase Dashboard:
- Go to **Edge Functions** -> `generate-reports`
- Configure a **Schedule / Cron** trigger
- Use **HTTP method** `POST`
- Add request header `Authorization: Bearer <SUPABASE_ANON_KEY>` (the anon key is a JWT, required because this function currently has JWT verification enabled)

## Required secrets / environment variables

The function reads these at runtime:
- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`

For Cron invocation headers (request-time auth):
- `Authorization: Bearer <SUPABASE_ANON_KEY>`

Set them in Supabase:
- **Project Settings** -> **Edge Functions** -> **Secrets**

Important:
- The **service role key** must never be embedded in the Electron app. It is server-side only.

## Manual run (backfill)

The function is idempotent and safe to re-run. It will backfill up to:
- 8 weekly periods (Fridays) if missing
- 3 monthly periods if missing

You can invoke it manually:
- **Edge Functions** -> `generate-reports` -> **Invoke**
- or via HTTP request (POST or GET)

## Verification (SQL)

After at least one successful run, you should see rows in `public.call_reports`.

Latest generated rows:

```sql
select
  period_type,
  period_start,
  period_end,
  scope,
  user_id,
  generated_at,
  (metrics->>'calls_total')::int as calls_total
from public.call_reports
order by generated_at desc
limit 50;
```

Most recent weekly and monthly **team** rows:

```sql
select *
from public.call_reports
where scope = 'team'
  and period_type in ('weekly','monthly')
order by period_type, period_start desc;
```
